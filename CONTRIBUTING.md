# Contributing to Taskflow Mobile

Thank you for considering contributing to the Taskflow Mobile project! We welcome contributions from everyone. Please follow these guidelines to help us manage the contributions effectively.

## Contribution Guidelines
1. **Fork the Repository**: Start by forking the repository and cloning it to your local machine.
2. **Create a Branch**: Create a new branch for your feature or bugfix. Use descriptive names for branches, e.g., `feature/cool-feature` or `bugfix/fix-something`.
3. **Make Your Changes**: Implement your changes in the new branch.

## Code Style
- Follow the existing code style of the project. Make sure your code is clean and well-documented.
- Use consistent indentation and naming conventions.

## Commit Conventions
- Write clear and concise commit messages. The format should be: `type(scope): subject`
  - **Type**: The type of change (e.g., feat, fix, docs, style, refactor).
  - **Scope**: Optional, what area of the codebase the change relates to (e.g., api, ui).
  - **Subject**: A short description of what the change does.

**Examples**:
- `feat(login): add login functionality`
- `fix(api): resolve user retrieval issue`

## Pull Request Process
1. **Push Your Changes**: Push your changes to your fork and branch.
2. **Open a Pull Request**: Navigate to the original repository and open a pull request. Fill out the template with relevant information.
3. **Review Process**: Your pull request will be reviewed by project maintainers. Feedback will be provided, and changes may be requested.
4. **Merging**: Once approved, your pull request will be merged into the main branch.

## Development Workflow
- Keep your fork up to date with the main repository to avoid merge conflicts.
- Make sure to run all tests before making a pull request.
- Follow any additional project-specific guidelines as outlined in this document or the repository documentation.

Thank you for contributing!
